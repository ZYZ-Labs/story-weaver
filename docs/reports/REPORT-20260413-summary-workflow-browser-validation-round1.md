## 验收方式

本轮不是只看 API。

使用了无头浏览器脚本，对已部署站点执行真实页面点击：

- 预注入登录态与当前项目
- 打开对应对象页
- 点击摘要工作流入口
- 在普通模式输入测试语料
- 点击 `让 AI 继续整理`
- 点击 `看看整理结果`
- 检查是否进入 `变化预览`

当前使用的项目：

- `旧日王座`
- `projectId = 28`

## 新发现

本轮没有发现新的后端阻塞问题。

第一轮脚本曾误把“页面上已有的提示文本”当成“AI 已返回草稿”，后来已修正脚本等待条件；修正后重新执行，四条路径全部通过。

这说明：

- 当前主链问题不在后端能力
- 也不在摘要工作流本身
- 现在更值得做的是普通作者体验减负，而不是继续补主链能力

## 后续建议

- 继续留在 `Phase 3.3`
- 把页面顶层的 `普通模式 / 专家模式` 显式切换再降一级
- 部署这轮轻量前端收口后，再做一次最终复验
