## 本轮目标

`Phase 10` 的目标不是再补新功能，而是把已经形成的平台主链变成：

- 可验证
- 可观测
- 可回放
- 可定位问题

一句话说：

- `Phase 9` 解决“新旧主链怎么一起活着”
- `Phase 10` 解决“出问题时怎么知道哪里坏了、能不能稳定复现”

## 本轮原则

- 先固定样本，再谈自动化
- 先补可观测，再补覆盖率
- 所有回放都要求：
  - 样本固定
  - 输出稳定
  - trace 可追
  - 状态可对比
- 不为了追求“全自动”而牺牲诊断可读性

## 范围内

- 固定章节样本回放
- 多 session trace 汇总与诊断
- 状态一致性检查
- 迁移正确性验证
- 页面级人工验收清单与可执行烟测
- 基础观测报表与报告模板

## 范围外

- 不在本阶段继续扩大功能范围
- 不在本阶段引入新的故事域对象族
- 不在本阶段做对外产品化发布

## 分阶段实施拆分

### `Phase 10.1` 样本集冻结与回放矩阵

目标：

- 冻结当前平台级固定样本
- 明确每个样本对应验证哪条链
- 建立章节级、项目级、页面级回放矩阵

交付物：

- 回放矩阵文档
- 固定样本章节清单
- 页面级验收清单

退出条件：

- 不再临时找样本做验证

### `Phase 10.2` trace / 状态一致性检查

目标：

- 补齐多 session trace 的检查口径
- 检查 `scene / handoff / chapter-review / patch / snapshot / event` 的一致性

交付物：

- trace 汇总规则
- 状态一致性检查规则
- 最小测试或脚本

退出条件：

- 关键主链可以判断“返回了，但是否合理”

### `Phase 10.3` 页面级烟测与人工验收模板

目标：

- 固定 `创作台 / 状态台 / 生成台 / 章节工作区 / 对象页` 的验收顺序
- 把目前零散的人肉验收变成制度化清单

交付物：

- 页面级烟测脚本或脚本基线
- 人工验收模板
- 结果记录模板

退出条件：

- 页面验收不再靠口头描述和记忆

当前补充：

- 已新增：
  - `docs/guides/GUIDE-20260424-phase10-page-acceptance-template-v1.md`

### `Phase 10.4` 迁移验收与阶段收口报告

目标：

- 汇总 `Phase 9` 迁移兼容与 `Phase 10` 回放结果
- 形成一份能支撑阶段收口的总报告

交付物：

- 迁移验收报告
- 回放验收报告
- 观测基线报告

退出条件：

- 平台主链具备“能重复验证”的收口证据

## 验证方式

- 固定章节接口回放
- 状态台 / 生成台 / 工作区人工验收
- 多 session trace 对照
- scene / handoff / review 一致性验证
- 迁移总览 / dry-run / compatibility snapshot 对照验证

## 风险

- 如果不先冻结样本，`Phase 10` 会重新掉回“临时找一章验证”的低效状态
- 如果没有 trace 和状态一致性检查，只看 `HTTP 200` 没有意义
- 如果不把页面级验收模板化，后面仍会重复踩“到底怎么测”的坑
