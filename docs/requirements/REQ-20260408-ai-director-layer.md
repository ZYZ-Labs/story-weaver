# AI 总导编排层

- Req ID: REQ-20260408-ai-director-layer
- Status: In Progress
- Created At: 2026-04-08 Asia/Shanghai
- Updated At: 2026-04-09 Asia/Shanghai

## 背景

当前项目已经具备章节、大纲、剧情、因果、世界观、知识片段、人物背包、背景聊天等上下文能力，但 AI 写作主链路仍以固定规则拼接 prompt 为主。系统会读取很多信息，却缺少一层独立的“判断者”来决定当前阶段该调什么模块、取什么字段、按什么优先级组织给写作模型。

现阶段的核心问题不是素材不足，而是编排层缺失。

## 目标

- 在现有写作链路前增加一个独立的 AI 决策层，作为总导/指挥中心。
- 决策层根据当前章节进度、有限章节参数和上下文状态，判断本轮写作应该使用哪些模块与约束。
- 决策层输出结构化决策结果，而不是直接产出正文。
- 决策层支持独立的 AI Provider 和 Model 配置，不与写作模型共用同一个 session。
- 引入 function call / tool calling，让决策层优先通过工具获取上下文，而不是依赖长 prompt。
- 保留现有编写层的 chat 式对话编写能力，不用决策层替代聊天写作。

## 范围内

- 决策层的独立入口和调用流程。
- 决策层可配置的 Provider / Model。
- 决策层的工具调用能力和工具白名单。
- 决策层的阶段判断、模块选择、权重/优先级选择。
- 决策层输出的结构化 `decision pack`。
- 写作层基于 `decision pack` 执行正文生成。
- 现有背景聊天提炼结果参与决策层输入，但不要求与决策层共用 session。
- 决策层失败时回退到当前固定规则聚合链路。

## 范围外

- 登录、安全和权限体系调整。
- 第一阶段直接新增大量新业务模块，如人物技能、人物好感度、势力关系等。
- 用决策层替代现有写作聊天面板。
- 第一阶段允许写作层自由调用全部业务工具。
- 第一阶段重做现有章节、大纲、剧情、因果、世界观模块的数据模型。

## 约束与默认假设

- 决策层和写作层必须是两个独立职责，不使用同一 session。
- 决策层尽量弱化 prompt，优先把规则转移到代码、工具 schema、JSON schema 和状态模型中。
- 决策层应优先输出结构化 JSON，不输出大段自然语言建议。
- 写作层的职责应收缩为“基于给定决策和上下文生成正文”，而不是继续承担大量上下文筛选责任。
- 写作层 chat 继续存在，仍作为人机协作写作入口；其稳定结论可经提炼后输入决策层。
- 第一阶段应优先编排已有模块，而不是继续堆新模块。

## 验收标准

- 系统可在写作前先由独立决策层判断章节阶段、写作模式和模块取用策略。
- 决策层可独立配置 Provider / Model，并与写作层配置解耦。
- 决策层可通过 function call 获取至少章节快照、大纲、剧情、因果、世界观、知识片段、人物状态等上下文。
- 决策层可输出稳定可解析的 `decision pack`，至少包含阶段、写作模式、选中模块、权重/优先级、关键约束和目标长度。
- 写作层能基于 `decision pack` 完成正文生成，同时保留原有 chat 式背景协作入口。
- 当决策层不可用或返回无效结果时，系统仍可退回当前固定规则聚合逻辑继续工作。

## 待确认问题

- 章节阶段模型要收敛成哪些有限状态。
- `decision pack` 的最终 JSON 字段和校验规则。
- 第一阶段开放给决策层的最小工具集合。
- 写作层是否允许读取极少量只读工具，还是完全只吃 `decision pack`。
- 决策层的长期状态如何落库，是否需要项目级/章节级快照。
