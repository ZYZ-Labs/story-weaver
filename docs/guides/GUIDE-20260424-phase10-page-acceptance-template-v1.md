## 用途

这份模板用于 `Phase 10.3` 的页面级人工验收，目标不是重新解释产品，而是把页面验收顺序和记录格式固定下来，避免每次联调都重新口头约定。

## 前置条件

- 已登录管理员账号
- 已切到目标项目
- 已确认当前部署版本包含待验收前端产物
- 后端接口已部署到与前端对应的版本

## 关键验收点

### 创作台

- 是否仍然是主入口
- 项目简报、下一步动作、章节骨架预览是否正常展示
- 跳转章节工作区是否顺手

### 状态台

- 章节状态、读者揭晓、迁移兼容、项目级 dry-run、一致性检查面板是否正常
- 当前章节无数据时，是否能正确提示推荐样本章节

### 生成台

- 多 session 编排预览是否完整
- candidate / brief / trace 是否可读
- `sceneId` 输入框与切换行为是否稳定

### 章节工作区

- 是否必须先显式生成镜头骨架，且未生成前不会假装已有多 scene
- 镜头列表是否来自 AI 骨架，而不是页面本地自动补齐
- 后续镜头是否可见但在前一镜头未接纳前不可越级选择
- 草稿流式生成是否有日志
- 切换 scene 后，草稿、日志、模型信息是否按 scene 隔离，不再互相覆盖
- 接受草稿后是否正确把当前镜头标记完成并解锁下一镜头
- `scene-2+` 的上一镜头摘要 / handoff 是否来自真实已接纳正文
- 删除镜头后的状态是否一致

### 对象页

- 是否统一采用 `Summary / Canon / State / History`
- 是否仍然存在字段墙或默认暴露过多系统细节

## 问题分级建议

- `P0`
  - 页面打不开、主链动作不可用、数据完全错位
- `P1`
  - 主链能走，但关键状态不对或明显误导
- `P2`
  - 排版、文案、局部交互缺陷

## 退出标准

- 所有主页面至少完成一轮人工验收
- `P0` 清零
- `P1` 要么清零，要么有明确延后说明
- 验收结果有文档记录，不再只存在于聊天上下文
